# Strict-Cost SDD — 设计 Spec

**状态：**提议的实验阶梯（非实现）。每一档仅在附带其门槛证据时才发布；任何门槛失败的档立即中止。
**目标：**最小化每次 plan 执行的美元开销。墙上时间不受约束；token 计数仅作为成本驱动因素而重要。
**硬性不变量：**质量。具体而言：`sdd-quality-reviewer-catches-planted-defect` 在 **N=5 次运行**（不是 1——单次运行门槛是本次战役最弱的方法论）上的通过率、`sdd-rejects-extra-features` 通过、所有端到端场景通过、与当前配置的盲评 A/B 交付物持平。任何质量回归都会终结该档，毫无商量。

## 钱花在哪里（2026-06-10 最终配置，go-fractals，约 $13/运行）

| 组件 | $ | 驱动因素 |
|---|---|---|
| Controller（会话模型，opus） | ~6-7 | ~150 轮 × 驻留上下文；prompt 免疫的轮次下限（46% 思考/叙述） |
| 实现者（sonnet，10-13 次调度） | ~5-6 | 实际工作；每个约 25 轮；每个约 13 次编辑前探索调用 |
| 任务审查者（sonnet，10 个） | ~1-1.5 | 带 package 时每次 3-9 轮 |
| 最终审查 + 修复 | ~1 | 带分支 package 时 6 轮 |

审查循环数（每次运行 2-4 次）是最大的运行间成本方差；循环主要由实现者错误解决的 plan 歧义引起。

## 判断护栏（与质量同为不变量）

**让机制变廉价，绝不让判断变廉价。**每一档必须枚举它把哪些决策移到了更廉价的模型，并展示每一项都是*机械的*——确定性的、可脚本化的、或事后可廉价验证的。判断留在最高层级或人类手中。SDD 中的判断点，明确如下：

- **BLOCKED / NEEDS_CONTEXT 处理**——诊断某个 subagent 为何卡住并选择补救
- **⚠️ "cannot verify from diff" 裁决**——controller 用跨任务上下文进行裁定
- **调度策展**——歧义消解和任务边界划分（实测承载作用：Task 5 的梯度方向注记防止了一次错误实现）
- **审查裁决和严重性校准**——什么是 Important vs Minor
- **审查循环裁定**——判定某发现是误报
- **升级到人类的识别**——知道 plan 本身是错的

任何一档若要把上述任何一项移到更廉价的模型，必须 (a) 重构以使该决策由昂贵模型在 plan 时一次做出，(b) 增加一条显式升级规则在执行时把它路由回去，或 (c) 终结。"廉价模型通常做对"不是验收证据——判断失败是罕见事件、高爆炸半径，且基本对通过/失败门槛不可见，这就是为何下方每一次层级变更除了 N=5 场景门槛外，还携带一份判断审计（对门槛运行中的每个判断点进行 session-resume 质询，与昂贵 controller 基线对照）。

## 论点护栏

SDD 的论点：**每个任务一个全新 subagent，带精心策展的上下文，按任务把关。**下方的各档必须保留它。调度时任务批处理（一次实现者调度处理多个 plan 任务）是**反论点**——它污染了全新上下文这一属性并粗化了门槛——因此被刻意排除在阶梯之外。通往相同调度经济性的、与论点兼容的路径是 plan 时的任务尺寸调整（L1）：如果 plan 定义了更少、尺寸更合适的任务，SDD 仍按每个任务运行一个全新 subagent。

## 阶梯（按预计 $/杠杆顺序）

### L1 — Plan 侧的清晰度（writing-plans 改动；估计每运行 -$1.5-3，外加方差降低）

**状态 2026-06-11（最终）：**端到端测试了引发；论断被重新归因。微测：约束头和 Interfaces 块能被确定性地引发（0→5/5、0→100% 任务、精确值）；尺寸调整是温和的且依赖规模（svelte 规模 9.4→8.4 个任务，fractals 规模无可动）。完整运行：一份引发的 plan 在 $6.34/$8.49 执行——但无指导对照（opus plan，完整代码）达到 $7.59/$7.73，落在该区间内。**成本收益属于 opus 编写的完整代码 plan；先前所有数字使用的手写 prose fixture plan 是不具代表性的，且执行起来约 2 倍昂贵。**该指导拥有的是保真度和方差：确定性约束传播（唯一一次引发运行的修复是版本下限捕获）、精确的跨任务接口、修复波 1 vs 2-4（对照 plan 交付了一个真实 Sierpinski bug，两次运行都得修）。writing-plans PR 在这些依据上主张，而非美元。草稿在 /tmp/sdd-exp/writing-plans-l1（分支 writing-plans-crisp）。

plan 处于每个成本的上游：任务数设定调度数；plan 歧义设定审查循环数；plan 完整度设定实现者探索。当前 writing-plans 为实现者成功而优化，而非执行经济性。待测改动：

1. **任务尺寸调整指导。**今天的 plan 会产出小到"create .gitignore"的任务——每个都花费一次完整调度 + 审查循环（约 $0.60-1.00 固定开销）。新增："A task is the smallest unit that carries its own test cycle and is worth a fresh reviewer's gate. Merge setup/config steps into the task that needs them; split only at boundaries where a reviewer could meaningfully reject." fractals 的 plan 会从 10 个任务降到约 7 个。验证：调度数下降、门槛守住、审查粒度仍能捕获植入缺陷。
2. **结构化的 `## Global Constraints` 章节**置于 plan 头部（版本下限、命名/文案规则、平台要求）。今天它们住在 design.md 的 prose 中，且仅当 controller 记得粘贴时才到达审查者（一个 `go 1.26.1` 下限违规之所以交付，正是因为没人粘贴）。固定的标题使它们可机械提取——`task-brief` 可自动追加到每份 brief（一次小脚本改动），完全移除一项 controller 职责。
3. **每任务 `Interfaces:` 行**（消费/产出、精确签名）。controller 当前按调度重新推导跨任务接口（它主要的合法"重述"），而实现者花费约 13 次工具调用重新发现上下文。规划者已经知道接口；每任务一行把工作移到只做一次的地方。
4. **每任务模型层级推荐**来自规划者（"mechanical / standard / judgment"）。规划者对 controller 当前按调度重新做出的 Model Selection 决策拥有最佳信息；controller 保留覆盖权。

验证：微测规划者输出形态（按指令设计学说的配方式），然后完整运行。注意 2026-06-10 的结果：当前 opus 无法引发 plan *占位符*——这些改动针对的是经济性和歧义，而非占位符卫生。

### L2 — Controller 层级（估计每运行 -$4-5；最大的单一杠杆，把关最严）

**状态 2026-06-11（最终）：**如预注册般在门槛处死亡——并附带了有用的剖析。侦察是正向的（$6.68/$8.05，n=2，机制干净）。完整电池拆分了判断面：新的 `sdd-escalates-broken-plan` 场景（显式 plan 自相矛盾；人类从不会主动提出）在 **sonnet 下 5/5 通过**（每运行 $1.02-1.37；opus 基线 2/2）——显式冲突会被升级。但植入缺陷电池决定性失败：在 sonnet controller 下，每任务质量门槛坍缩为 plan 合规倡导（"no assertion, as required"被列在 Strengths 下），缺陷在 4/5 运行中交付（确定性检查），且只有层级锁定的 opus 最终审查者捕获它——而同样的 sonnet 层级审查者在 opus controller 下 5/5 标记了它。廉价 controller 处理显式升级；它们吸收了隐式的权威-vs-质量裁定。一个可能的 L2b（离散规则："a reviewer finding that conflicts with the plan's text is the human's decision — escalate it"）会把失败的判断路由到曾成立的升级行为中。

**L2b 于 2026-06-11 测试（E35/E36，evals `docs/experiments/2026-06-11-build-loop-autoresearch.md`）：**改善了 opus 栈，未能拯救 sonnet 档。两条规则：一条审查者绊线（plan 强制的缺陷即是一项发现——Important、标注 plan-mandated；由人类决定）和一条 controller 升级规则（plan-mandated 发现像任何 plan 矛盾一样交给人类）。在冻结的 sonnet 编写输入上微测：0/6 → 6/6 标注发现。完整电池：opus controller 2/2 内化了规则，作为自称的后备捕获了其审查者的遗漏并升级以获得批准的修复（4241 临时行为被结构化）；升级健全性 2/2 未破。Sonnet controller：1/5 完整通过——复述把绊线从调度中丢弃（2/5 传输），仅传输在实时中并不触发它（在审查者的工具读取间被一次性读取稀释；调度内的放置被驳斥为变量），且无 sonnet controller 展示后备行为；1/5 交付了缺陷。L2b 规则是 opus 栈的候选提交。一个面向 sonnet 档的未来 L2c 会把 SKILL.md 的约束配方（sonnet 逐字传输的唯一通道）与 plan-mandated 发现的强制输出格式槽位（该骨架经受住每次观察到的复述并在编写时被咨询）配对；未测。原始侦察笔记如下。

**侦察（已被取代）：**
Sonnet-controller 运行（claude-sonnet coding-agent）：全部门槛绿色，在 **$6.68 和 $8.05** / 31-41 分钟（组合区间 $11.67-14.84），token 落在组合区间内——无廉价 controller 的轮次膨胀。26/26 和 31/31 调度模型显式，带有比 opus controller 更重（且合理）的 haiku 分层；审查循环、每任务 Important→修复→复审、omnibus-fixer 规则在两次运行中都被遵循；运行 1 的 controller 在复审前捕获了一个修复器副作用（`go mod tidy` 移除了 cobra）——真实的裁定，不是静默吸收。但两次运行都未浮现 BLOCKED/⚠️ 事件（升级点从未被压测），且最终审查运行在 sonnet 上而非最强层级。下方的 N=5 质量门槛 + 完整判断审计在任何 skill 改动前仍是强制性的。

controller 占了一半费用，仅因为它继承了会话模型。其轮次下限是 prompt 免疫的，所以杠杆在于每轮单价——但 controller 也是大多数判断点所在之处，所以本档以判断优先设计：

1. **主要形式——判断前移、机制变廉：**昂贵模型在 plan 时做判断密集型工作（L1 的 Interfaces 行、歧义消解、每任务约束——即把调度策展预写进 plan）。中层执行会话随后运行一个真正机械的循环：提取 brief、调度、运行脚本、路由裁决。skill 中的显式升级规则：在 BLOCKED 时、在任何 ⚠️ 条目时、在疑似误报时、或在 plan 尚未回答的任何事项上，廉价 controller 停下并升级（给人类，或给一次全新的昂贵模型咨询调度）——它绝不独自裁决判断。
2. **超出标准 N=5 的门槛：**一份判断审计——门槛运行中的每个 BLOCKED/⚠️/裁定事件都通过 session-resume 被质询，并对照 opus-controller 基线如何处理同类事件进行打分；任何静默吸收的判断调用（廉价 controller 裁决了本应升级的事项）无论场景裁决如何都终结该档。
3. **用户权威保留：**skill 建议、绝不强制执行会话层级。

本次战役的告诫：廉价模型的轮次膨胀是在多步*工作*上测量的，而非调度循环；中层 controller 是否能维持约 150 轮正是实验要确定的一部分。

### L3 — 审查者层级（估计每运行 -$0.7-1；最可能在判断护栏上死亡的档）

**状态 2026-06-11：**如预注册般死亡。植入缺陷 ×5 配强制 haiku 任务审查者：2 通过 / 1 不确定 / 2 失败（基线 5/5）；每任务 haiku 在正确严重性下干净地标记了 10 个植入缺陷中的 0 个——1 个发现但被降级并带有被禁止的精确理由，9 个遗漏或被合理化（DRY 被赞为 YAGNI；assert-nothing 测试被称为 plan-compliant）。廉价审查者通过*倡导*缺陷而失败；通过的运行仅靠 controller 冗余或最终审查存活。记录在实验日志中，Batch A-E。在没有结构性不同设计的情况下不要重提。

带 package 的审查者在机械上接近单步（平静时 3 轮 / 1 次 Read），这使中层下限的原始轮次膨胀理由失效——但审查彻头彻尾是判断：严重性校准、spec 裁决、知道什么不该标记。机械上的廉价并不使决策变机械。仅在配完整判断电池时测试 haiku-with-package：植入缺陷 ×5、一次严重性校准检查（植入 Minor-vs-Important 对；校准失误终结该档）、以及在该层级上重新测量的逃生舱方差。先验预期：本档死亡，而这是一个好结果——它把"我们怀疑廉价审查者很差"转化为记录在案的证据。

### L4 — 驻留上下文节食（估计每运行 -$0.5-1）

- `task-brief --list` 模式：controller 读任务标题 + Global Constraints，从不读完整 plan（plan 正文已通过 brief 交付）。
- 报告从 15 行裁剪到 8 行。
- SKILL.md 压缩一轮（本周新增的每个章节都按编写配方密度重新论证；Codex 在一次长会话中付出约 10k 字符 × 约 500 次重读）。

### L5 — 重新争辩（明确标记，被维护者否决或反论点）

为完整性记录；每一项在实验前都需要 Jesse 的明确反转：
- **范围限定的复审**（验证修复 + 回归扫描而非完整复审）：2026-06-09 否决；每运行至多值约 $0.50。
- **调度时任务批处理**：反论点（见护栏）。L1.1 是被批准的形式。

## 预算和排序

L1 和 L2.1 相互独立——先都跑（约 $80：微测 + 2×5 次运行门槛 + A/B）。L3 在 L2 稳定 controller 之后（审查者行为依赖调度质量；约 $25——植入缺陷运行每次 $2-3）。L4 最后（便宜，但在栈完成后重新过一次门槛；约 $30）。完整阶梯配诚实的 N=5 门槛总计 ≲ $150。若每档都通过其门槛的预期终态：**fractals 上每运行 $5-7（从 $12-15）**；若判断敏感的档（L2 超出其主要形式、L3）如预期死亡，则 **每运行 $8-10**——这是诚实目标，因为护栏按构造把判断定价在美元之上。
