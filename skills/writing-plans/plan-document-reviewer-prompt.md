# Plan 文档审查者 Prompt 模板

当 dispatch plan 文档审查者 subagent 时使用此模板。

**目的：** 验证计划是完整的、匹配 spec 的，并有适当的任务分解。

**Dispatch 时机：** 完整计划已编写。

```
Task tool (general-purpose):
  description: "Review plan document"
  prompt: |
    你是 plan 文档审查者。验证此计划是否完整并准备好实施。

    **待审查的计划：** [PLAN_FILE_PATH]
    **参考 spec：** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 寻找内容 |
    |----------|------------------|
    | 完整性 | TODOs、占位符、未完成的任务、缺失的步骤 |
    | Spec 对齐 | 计划覆盖 spec 需求、无重大范围蔓延 |
    | 任务分解 | 任务有清晰的边界、步骤可执行 |
    | 可构建性 | 工程师能否遵循此计划而不受阻？ |

    ## 校准

    **仅标记会在实施期间导致实际问题的问题。**
    实施者构建错误的东西或受阻是问题。次要措辞、风格偏好和"锦上添花"的建议不是问题。

    除非有严重缺口 — 缺失 spec 中的需求、矛盾的步骤、占位符内容，或任务太模糊而无法执行，否则批准。

    ## 输出格式

    ## Plan 审查

    **状态：** Approved | Issues Found

    **问题（如有）：**
    - [任务 X，步骤 Y]：[具体问题] - [为什么对实施很重要]

    **建议（建议性，不阻止批准）：**
    - [改进建议]
```

**审查者返回：** 状态、问题（如有）、建议
